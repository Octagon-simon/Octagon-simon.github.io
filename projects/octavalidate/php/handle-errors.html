SERVER-SIDE VALIDATION ERRORS

It is really necessary to handle server-side validation errors. This is because the user needs to know what validation test(s) failed and the one(s) that passed.

Immediately you invoke the validate method, the library loops through the POST ARRAY, executes validation tests, and stores errors if any test fails.

To retrieve these errors, you would do something like this;

if($myForm->validate($valRules) === true){
  //process form data here
}else{
  //return errors
  print_r($myForm->getErrors());
}

Make sure to invoke the getErrors() method, only when the validate method returns false!

Now that we have retrieved the errors, we need to call a JavaScript function to help us display these errors to the end-user.

HELPER SCRIPT

There is a helper script containing the JavaScript functions needed to display these errors that is attached to the library.

Navigate to library_root_folder/frontend/helper.js to locate this helper script, then include the functions to your project.

showErrors

Invoke this function and pass in the error object from the method getErrors() to insert errors to the form.

removeErrors

Call this function and pass in a formID to remove any error(s) inserted by the showErrors function.

AJAX

If you're using Ajax, within the error method, you need to call the showErrors function and pass in the error object for the function to insert errors into the form, then within the success method, you need to call the removeErrors function and pass in the formID to remove any errors present in the form. 

 $.ajax({
    ... 
  success: function (res) {
     //pass in the form id to this function
     removeErrors(formId);
  },
  error: function (xhr) {
      if (xhr.readyState === 4) {
         //pass in the errors array to this function
         showErrors(JSON.parse(xhr.responseText));
      }
  }
    ...
})

Remember that not all failed http requests are to be treated as validation errors. You can set a response code to make your script understand that the failed request is a validation error.

PHP DIRECT

If your form is submitted to the page itself, you can easily call the showErrors function and pass in the error object for the function to insert errors into the form.

if($myForm->validate($valRules) === true){
    //process form data here
}else{
    //return and display errors
    print('<script>
    document.addEventListener("DOMContentLoaded", function(){
       showErrors(' . $myForm->getErrors() . ');
    });</script>');
}

HELPER SCRIPT FOR FRONTEND LIBRARY

Well if you're using this library for frontend validation, you do not need to include the helper script because the functions contained in that script is already included in the library for frontend validation.

//create new instance
const myForm = new octaValidate('FORM_ID');
//invoke this method to show server-side validation errors
myFrom.showBackendErrors('ERROR_OBJECT');
//invoke this method to remove errors present in a form
myForm.removeBackendErrors('FORM_ID');
