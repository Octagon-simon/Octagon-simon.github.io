VALIDATION RULES

Validation rules allows you to perform validations on form inputs using the rule tiles provided. 

//syntax
$valRules = array(
  "FORM_INPUT_NAME" => array(
    ["RULE_TITLE", "CUSTOM_ERROR_MESSAGE"]
  )
);

Here, you can find all validation rules and their description.

Rule Title Description
R Required
EMAIL Email
ALPHA_ONLY Letters Only
ALPHA_SPACES Letters and Spaces
ALPHA_NUMERIC Letters with Numbers
LOWER ALPHA Lowercase letters
UPPER_ALPHA Uppercase letters
DIGITS Numbers or Digits
PWD Password
URL URL
URL_QP URL with Query Parameters
DATE_MDY Date in the format MM/DD/YYYY
USERNAME Username
TEXT General Text
Can't see a rule that you need for your form? Use the custom rule api to build your own validation rule.
Learn more

The TEXT validation rule supports these special characters (. , / () [] & ! '' "" : ; ?) and this means that any character outside the supported set will be counted as invalid!

Follow the syntax below to define a validation rule for a form input name. 

//sample validation
$valRules = array(
  "username" => array(
    ["R", "Your username is required"]
 ),
  "email" => array(
    ["R", "Your Email Address is required"],
    ["EMAIL", "Your Email Address is invalid!"]
  )
);

After defining the rules, pass it to the validate method to begin validation. Learn more

If you don't provide a custom error message, the script will use the default one available.
