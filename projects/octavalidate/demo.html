On The Demo Form

  • Strict Mode is enabled

{ strictMode: true, strictWords : ["null", "NaN", "undefined"] }

  • Reserved Keywords

With strict mode enabled, users are not allowed to submit reserved keywords.

Aside from the default strict words, you are allowed to provide a custom set of words to check against.

Try to provide null, NaN or undefined as a value, then submit.

  • Input value is Trimmed

With strict mode enabled, extra white space at the start and at the end of an input element is removed.

  • successBorder is enabled

{ successBorder: true }

The input element should automatically set a green border if validation test was passed.

  • callBacks are defined

validateCallBack(success_callback, error_callback);

With the new callback feature, you can define a function that will be invoked if there are no errors or if there are errors, then pass it as an argument to the validateCallBack() method.

  • validation status is returned

myForm.status();

Invoke this method anytime to see how many validation errors on the form.

With the callback and status methods, you can alert the user in real-time of the number of validation errors on the form.
