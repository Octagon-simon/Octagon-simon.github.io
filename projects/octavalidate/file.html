FILE VALIDATION

To mark a file input as required, use the R validation rule.

Here are some of the attributes needed to validate a file.

  • accept - Use this attribute to list out the file types allowed for upload
  • size (2mb) single - This means that the file provided must be 2mb in size
  • minsize (5mb) single - This means that the file provided must be up to 5mb or more.
  • maxsize (5mb) single - This means that the file provided must be 5mb or less.
  • totalsize (2mb) multiple - This means that all files provided must have a size equal to 2mb.
  • totalminsize (5mb) multiple - This means that all files provided must have a size that is up to 5mb or more.
  • totalmaxsize (5mb) multiple - This means that all files provided must have a size that is 5mb or less.

multiple means that the file upload type must have a multiple attribute to support more than one file.

Attribute Possible values File upload Type Custom message
accept .png, .jpg, .jpeg multiple & single ov-accept:msg
size 2000 or 2000bytes, 5mb, 10gb, 15pb, 20tb etc single ov-size:msg
minsize 2000 or 2000bytes, 5mb, 10gb, 15pb, 20tb etc single ov-minsize:msg
maxsize 2000 or 2000bytes, 5mb, 10gb, 15pb, 20tb etc single ov-maxsize:msg
totalsize 2000 or 2000bytes, 5mb, 10gb, 15pb, 20tb etc multiple ov-totalsize:msg
totalminsize 2000 or 2000bytes, 5mb, 10gb, 15pb, 20tb etc multiple ov-totalminsize:msg
totalmaxsize 2000 or 2000bytes, 5mb, 10gb, 15pb, 20tb etc multiple ov-totalmaxsize:msg
To prevent the browser from using its inbuilt accept validation on your form, add the attribute novalidate to the form tag.
In as much as the size attributes supports a single value like 200, if you don't provide any digital storage from bytes up to terabytes (eg 200mb), we will assume that the value provided is in bytes.

Note that size, minsize & maxsize were built for single file upload. If you use any of them on multiple files upload, it will make the script to check if each of the files provided meets the required size.

DEMO FORM

<!-- single file upload -->
<input id="inp_single_file" type="file" accept=".png,.jpg,.jpeg" maxsize="2mb" octavalidate="R">
<!-- multiple file upload -->
<input id="inp_multi_file" type="file" accept=".mp3,.ogg" totalmaxsize="10mb" octavalidate="R" multiple>

NOW, TRY TO BREAK THROUGH! 😎

.jpeg, .jpg, or .png file is only allowed, maximum size of file is 2MB
.mp3 or .ogg files only allowed, maximum size of all files is 10MB.
